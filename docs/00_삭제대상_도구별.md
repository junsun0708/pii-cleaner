# 도구별 PII 삭제 대상

## 1. Jira (Atlassian Cloud / Data Center)

| 대상 | 위치 | 처리 방법 | API/CLI |
|---|---|---|---|
| 계정 | Atlassian 계정 | 비활성/익명화 | `POST /admin/v1/users/{accountId}/anonymize` (GDPR API) |
| 이슈 본문 | description | 정규식 redact 또는 수정 | `PUT /rest/api/3/issue/{key}` |
| 이슈 댓글 | comment.body | 수정 또는 삭제 | `DELETE /rest/api/3/issue/{key}/comment/{id}` |
| @mention | description/comment 안 `[~accountId]` | 익명 ID 로 치환 | (위와 동일) |
| assignee / reporter / watcher | 이슈 필드 | 익명 계정 재할당 | `PUT /rest/api/3/issue/{key}` |
| 첨부 파일 | attachment | 삭제 | `DELETE /rest/api/3/attachment/{id}` |
| changelog | 활동 이력 | GDPR 익명화 (수정 불가) | Atlassian Anonymize |
| 검색 인덱스 | Lucene | 익명화 후 자동 재색인 | (자동) |
| Confluence (연결 시) | 페이지·댓글·공간 | 별도 작업 | `/wiki/rest/api/...` |

⚠ Atlassian Cloud 의 GDPR 익명화는 비가역. 사전 export 필수.

## 2. Bitbucket (Cloud / Server)

| 대상 | 위치 | 처리 방법 | 도구 |
|---|---|---|---|
| 커밋 author / committer | `.git` 의 `user.name`/`user.email` | 히스토리 재작성 | `git filter-repo --mailmap` 또는 BFG |
| 커밋 메시지 | git log msg | 재작성 (force-push) | `git filter-repo --commit-callback` |
| 코드 안 자격증명·PII | `.env`, `application.yml`, `*.sql` | 정규식 redact + git history 재작성 | `git filter-repo --replace-text` |
| PR description / comments / inline | Bitbucket UI | 삭제 또는 수정 | `DELETE /2.0/repositories/{ws}/{repo}/pullrequests/{id}/comments/{cid}` |
| 사용자 계정 | Atlassian 계정 | Jira 와 공유 — 한번에 익명화 | (위 Jira 와 동일) |
| SSH 공개키 | 사용자별 등록 키 | 삭제 | `DELETE /users/{username}/ssh-keys/{id}` |
| App passwords / API tokens | 사용자 보안 페이지 | 폐기 | `DELETE /2.0/users/{username}/app-passwords/{id}` |
| Webhook URL | 레포/워크스페이스 | 외부 봇 무효화 검토 | `DELETE /2.0/repositories/{ws}/{repo}/hooks/{uid}` |

⚠ 커밋 히스토리 재작성 = `git push --force` 필요. 다른 개발자 로컬 영향. 사전 공지 + 미러 백업.

## 3. Slack

| 대상 | 위치 | 처리 방법 | API |
|---|---|---|---|
| 사용자 프로필 | 실명·휴대전화·사진·status·display name | 익명 값으로 치환 | `users.profile.set` |
| 공개 채널 메시지 | 본인 또는 워크스페이스 owner | 삭제 | `chat.delete` |
| DM (1:1, 그룹) | 본인만 삭제 가능 / Enterprise Grid 는 Discovery API | 사용자 자가 삭제 또는 Compliance | `chat.delete` (자기 메시지) / `discovery.chats.delete` |
| 첨부 파일 | files API | 삭제 (외부 공유 링크도 무효화) | `files.delete` |
| 봇 메시지 / Webhook 페이로드 | 자동화 안 PII | 봇 토큰으로 본인 메시지 삭제 | `chat.delete` |
| Discovery 익스포트 보관본 | Compliance 영역 | GDPR 정책으로 영구 삭제 요청 | Vendor 문의 |
| 앱 토큰 / Bot Token / Webhook URL | 워크스페이스 앱 설정 | 폐기/재발급 | `admin.apps.uninstall` 등 |

⚠ 표준 Slack 플랜은 DM 일괄 삭제 API 없음. Enterprise Grid + Discovery API 필요.

## 4. Microsoft 365

### 4-1. Entra ID (구 Azure AD)
| 대상 | 처리 |
|---|---|
| 사용자 계정 | 비활성 후 삭제 (소프트 삭제 30일 → 영구) |
| 그룹 멤버십 | 그룹에서 제거 |
| MFA 등록기기·디바이스 등록 | 등록 해제 |
| 로그인 로그 / 감사 로그 | 보관기간 정책 적용 (기본 30일~) |
| Conditional Access 로그 | 동일 |
API: Microsoft Graph `/users/{id}`, `/auditLogs/...`

### 4-2. Exchange (Outlook)
| 대상 | 처리 |
|---|---|
| 사서함 | export (PST) 후 비활성/삭제 |
| 받은편지·보낸편지·임시·삭제·휴지통·복구가능항목 | 모두 정리 (단계별) |
| 연락처 / 일정 / 참석자·메모 | 삭제 |
| 위임 권한·자동 응답·메일 규칙 | 해제 |
| In-Place Archive | 보관함 export 후 정리 |
| Litigation Hold 적용 사서함 | **Hold 해제 후 삭제** (해제 권한 = 컴플라이언스) |
API: Microsoft Graph `/users/{id}/messages`, `/users/{id}/contacts`, Exchange Online PowerShell

### 4-3. Teams
| 대상 | 처리 |
|---|---|
| 1:1·그룹 채팅 | 사용자 본인 삭제 또는 Compliance Search-and-Purge |
| 채널 게시물·댓글 | 채널 owner 가 삭제 |
| 회의 녹화·트랜스크립트·화이트보드 | SharePoint/OneDrive 의 미디어 파일 삭제 |
| 통화/회의 로그 | Teams Admin Center 활동 보고서 정책 |
API: Microsoft Graph `/chats/...`, `/teams/.../channels/.../messages/...`, Security & Compliance Center

### 4-4. OneDrive
| 대상 | 처리 |
|---|---|
| 사용자 OneDrive 콘텐츠 | 퇴사 30일 후 자동 삭제 옵션 / 또는 export 후 즉시 삭제 |
| 공유 링크·외부 게스트 접근 | 링크 무효화, 게스트 제거 |
API: Microsoft Graph `/users/{id}/drive/...`

### 4-5. SharePoint
| 대상 | 처리 |
|---|---|

| 라이브러리 파일 (개인 파일) | 삭제 |
API: Microsoft Graph `/sites/...`, SharePoint REST

## 도구별 인증·권한

| 도구 | 인증 | 필요 권한 |
|---|---|---|
| Jira | API Token (사용자) 또는 OAuth 2.0 (3LO) | `manage:jira-data-provider` 또는 Site Admin |
| Bitbucket | App Password 또는 OAuth | Workspace Admin (히스토리 재작성 시 repo admin) |
| Slack | User OAuth Token + Bot Token (필요시 Enterprise Discovery) | `users:read`, `chat:write`, `files:read/write`, `admin.users:write` |
| MS Graph | Application Permissions (Client Credentials) | `User.ReadWrite.All`, `Mail.ReadWrite`, `Files.ReadWrite.All`, `Chat.ReadWrite.All`, `Group.ReadWrite.All`, `Directory.ReadWrite.All` 등 |
