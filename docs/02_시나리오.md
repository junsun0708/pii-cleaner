# 시나리오별 처리 정책

같은 도구라도 「누구의 어떤 데이터를 왜」 정리하느냐에 따라 우선순위가 달라진다.

## 시나리오 A — 퇴사자 정리 (가장 빈번)

### 목표
법정 보관 유지 + 계정·DM·개인 자료 정리

### 단계
1. **계정 비활성** (모든 도구) — 즉시 접근 차단
2. **개인 자료 export** — OneDrive, Outlook 사서함, Jira/Bitbucket 활동 → 관리자 보관
3. **DM/사적 채팅** — Slack DM, Teams 1:1 → Compliance Search-and-Purge
4. **공유 자료 안 멘션·assignee** — 익명 계정 재할당 또는 익명화
5. **인증 정리** — SSH 키, API 토큰, App password, MFA 등록기기 모두 폐기
6. **30일 유예 후 영구 삭제** (Azure AD 정책 + Atlassian GDPR Anonymize)
7. **회계·계약 관련 데이터는 익명화 후 보관** (10년)

### 우선순위
| 단계 | 도구 | 시점 |
|---|---|---|
| 1 (즉시) | Entra ID, Slack, Jira/Bitbucket | 퇴사 당일 |
| 2 (24h) | OneDrive/Outlook export | 퇴사 1일차 |
| 3 (7일) | DM·1:1 채팅 Search-and-Purge | 인수인계 종료 후 |
| 4 (30일) | 영구 삭제 (Azure AD), Atlassian Anonymize | 유예기간 종료 |

## 시나리오 B — 외부 고객·파트너 정보 정리

### 목표
고객 요청 (Right to be Forgotten) 또는 계약 종료 후 개인정보 삭제

### 단계
1. **고객 식별** — 이메일·이름·연락처로 4개 도구 grep
2. **법정 보관 의무 확인** — 회계·세무·계약 (10년) 제외
3. **메일·채팅 안 언급 식별** — Slack/Teams 대화, Jira 댓글
4. **첨부 파일 정리** — 계약서·견적서 안 PII 일부만 redact
5. **외부 공유 링크 무효화** — OneDrive/SharePoint 공유, Slack files
6. **익명화** — 회계 추적 가능 ID (예: customer_12345) 로 치환

### 도구별 우선순위
- Slack/Teams DM·채팅 (가장 분산)
- Outlook 메일 (보낸·받은 양방향)
- Jira/Confluence 이슈·페이지
- Bitbucket 코드 안 하드코드 (드뭄, 검증)

## 시나리오 C — 특정 기간 데이터 일괄 정리

### 목표
오래된 (예: 5년 초과) 데이터 일괄 정리 — 스토리지 절감, GDPR 「최소수단」

### 단계
1. **법정 보관기간 매트릭스 적용** — 데이터 종류별 가능 시점 산출
2. **회계·계약·인사 분리** — 보관 필요 데이터 격리
3. **Litigation Hold 확인** — 분쟁 중인 데이터 제외
4. **자동화 정책 적용** — 도구별 Retention Policy 설정
5. **Audit Log** — 처리 결과 기록 (그 자체로 증빙)

### 자동화 정책 권장
- Outlook: Retention Policy (예: 5년 후 자동 삭제)
- Teams: Channel Retention (1년)
- OneDrive: 게스트 링크 만료 (30일)
- Slack: Message Retention (Enterprise — 1년)
- Jira: 이슈 아카이브 (3년)

## 시나리오 D — 코드 안 하드코드 정리 (Bitbucket 특화)

### 목표
실수로 커밋된 비밀번호·API 키·개인 이메일 제거

### 단계
1. **git 히스토리 grep** — `git log -p | grep -iE 'password|secret|token|@gmail.com|010-[0-9]'`
2. **gitleaks / truffleHog 같은 도구 활용**
3. **`git filter-repo` 또는 BFG** 로 히스토리 재작성
4. **force-push + 팀 공지** — 모든 로컬 clone 재설정
5. **노출된 자격증명은 즉시 회전** (rotate) — 히스토리 정리 전에라도

⚠ 히스토리 재작성은 모든 팀원 영향. 사전 공지 + 미러 백업 필수.
