# 법적 고려사항

## 1. 한국 — 개인정보보호법

- **제21조 (개인정보 파기)**: 보유기간 경과·처리목적 달성 시 지체없이 파기. 단 다른 법령에 의해 보존 필요 시 분리 보관.
- **제29조 (안전조치 의무)**: 파기 시 복구 불가능한 방법.
- **개인정보 처리방침** 에 명시된 보유기간 우선.

## 2. EU — GDPR (Atlassian/Microsoft/Slack 모두 준수)

- **Right to be Forgotten** (제17조): 정보주체 요청 시 삭제. 단 법적 의무·공익·표현의 자유 시 거부 가능.
- **Pseudonymization** (제4조 5항): 추가 정보 없으면 식별 불가능한 처리. 「삭제」 보다 권장되는 경우 多.

## 3. 법정 보관기간 (한국 기준)

| 데이터 | 법령 | 기간 |
|---|---|---|
| 회계장부·증빙 | 상법 §33, 국세기본법 §85의3 | **10년** |
| 근로계약·근태 | 근로기준법 §42 | **3년** |
| 인사기록 (퇴직자) | 통상 사규 | 3~5년 (사업 결정) |
| 계약서·거래 관련 메일 | 상법 §33 | **10년** |
| 통신·접속 로그 (통신비밀보호법 적용 시) | 통비법 §15의2 | **3개월~12개월** |
| 의료·환자 정보 | 의료법 §22 | 10년 (보건의료 분야) |

→ **회계·계약 관련 이메일·문서는 무조건 삭제 X**. 익명화 또는 **법정 보관기간 종료 후 삭제**.

## 4. SaaS 의 데이터 보관 정책

### Atlassian (Jira/Bitbucket)
- 계정 비활성 → 14일 유예 → 안내 → 익명화 가능
- **GDPR Anonymize API 는 비가역**. 사전 export 필수
- Cloud 백업: 회사 plan 에 따라 다름

### Slack
- 표준 플랜: 삭제 즉시 시스템 보관 90일 후 영구 삭제
- Enterprise Grid + Compliance: 영구 보관 (Hold) 가능, 별도 정책으로 정리
- **Discovery 익스포트** (Enterprise) 보관본은 별도 vendor 요청

### Microsoft 365
- **Soft Delete (30일)** → Recycle Bin → **Hard Delete**
- **Retention Policy / Litigation Hold** 적용 시 삭제 거부 — 해제 권한 (eDiscovery Admin) 필요
- 사서함 삭제 후 「복구 가능 항목 (Recoverable Items)」 폴더 14일 보관
- Azure AD 사용자: 30일 보존 → 영구

## 5. eDiscovery / Litigation Hold

소송·감사 진행 중이면 모든 도구에 「**Hold**」 적용 가능. Hold 데이터는:
- API 삭제 시도 → **403 Forbidden** 또는 사일런트 무시
- 해제 권한 = 컴플라이언스/법무 팀
- **반드시 해제 확인 후 삭제 진행**

## 6. 백업본 처리

- SaaS vendor 의 자체 백업 (DR 용) 은 회사가 직접 삭제 못 함
- **DPA (Data Processing Agreement)** 또는 GDPR 요청으로 영구 삭제 보장
- 일반적으로 vendor SLA = 30~90일 내 백업본 정리

## 7. 권장 절차

```
1. 컴플라이언스 팀 사전 협의
2. 법정 보관 vs 삭제 가능 분리 (회계/계약 = 보관)
3. eDiscovery / Litigation Hold 해제 확인
4. dry-run — 영향 범위 산출
5. export (백업) — 법적 요건 대비
6. 익명화 우선, 불가시 삭제
7. vendor 측 백업본 영구 삭제 요청 (필요시)
8. 처리 결과 로그 보관 (역설적 — 처리 사실 자체는 기록 필요)
```
