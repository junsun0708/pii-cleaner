# DM 일괄 삭제 — 단계별 절차

`mvp_slack_self_delete/slack_self_delete.py` 를 사용해서 특정 사용자와의 DM 안 본인 메시지를 일괄 삭제하는 전체 흐름.

## 적용 범위 (권고)

| 대상 | 권장 |
|---|---|
| ✅ **1:1 DM** | 본인 흔적 정리 (퇴사·실수 메시지). 본인 메시지만, Slack 정책상 상대방 메시지는 못 건드림 |
| ⚠ **그룹 DM (3명+)** | 다른 참여자 양해 받고 진행 |
| ❌ **공개·비공개 채널** | 권장 안 함 — 「공동 자산」, 팀 컨텍스트 손실, 컴플라이언스 위반 가능 |

→ **상세 권고는 `00_삭제대상_도구별.md` + `01_법적고려사항.md` 참고**

---

## 0. 사전 확인

### 0-1. 회사 워크스페이스 정책
1. 워크스페이스 좌상단 이름 클릭 → 「설정 및 관리」 → 「설정 및 권한」 → 「Permissions」
2. **「Members can manage apps」** 정책 확인:
   - "Anyone" → 자유 (Case A)
   - "Only owners and admins" 또는 "Members can request" → **관리자 승인 필요** (Case B)

### 0-2. Litigation Hold / Retention Policy
- 분쟁 진행 중이면 삭제 거부 — 컴플라이언스 팀 사전 협의
- 회계·계약 관련 DM 은 법정 보관 5~10년 (`01_법적고려사항.md` 참조)

---

## 1. Slack App 생성

1. https://api.slack.com/apps 접속 (본인 Slack 계정 로그인)
2. 우측 상단 **「Create New App」** → **「From scratch」**
3. App Name: 아무거나 (예: `my-cleaner`)
4. Workspace: 회사 워크스페이스 선택
5. **「Create App」**

> 워크스페이스 드롭다운에 회사가 안 보이면 = Case B (관리자 차단). 관리자에게 「내가 만든 App 승인 부탁」 요청 또는 「토큰 직접 발급 부탁」 요청.

---

## 2. 권한 (Scope) 추가

좌측 메뉴 **「OAuth & Permissions」** → **「Scopes」** → **「User Token Scopes」** 영역 (Bot Token Scopes 아님!) 에 추가:

### 필수 (8개)
| Scope | 용도 |
|---|---|
| `chat:write` | 본인 메시지 삭제 |
| `im:history` | 1:1 DM 메시지 조회 |
| `im:read` | DM 채널 메타 조회 (`conversations.info` 호출용. 없어도 동작 가능) |
| `im:write` | DM 채널 열기 (`--user` 옵션으로 상대방 지정 시) |
| `users:read` | 사용자 ID 조회 (`--user` 사용 시) |
| `users:read.email` | 이메일로 사용자 조회 (`--user hong@example.com`) |
| `files:write` | 첨부 파일 삭제 (`--delete-files` 사용 시) |
| `mpim:history` | 그룹 DM 메시지 조회 (그룹 DM 대상 시) |

### 옵션 (채널까지 정리하려면)
| Scope | 용도 |
|---|---|
| `channels:history` | 공개 채널 조회 (⚠ 비권장) |
| `groups:history` | 비공개 채널 조회 (⚠ 비권장) |
| `channels:read` / `groups:read` | 채널 메타 조회 |

---

## 3. 워크스페이스 설치 + 토큰 발급

1. **「OAuth & Permissions」** 페이지 상단 **「Install to Workspace」** 클릭
2. 권한 동의 화면 → **「허용」**
3. 결과:
   - **Case A 자유 정책**: 즉시 토큰 발급 ✅
   - **Case B 관리자 승인 필요**: "Pending Approval" → 관리자 Slack 알림 자동 발송 → 승인 (1~2일) → 토큰 발급
4. **「User OAuth Token」** = `xoxp-1234567890-...` 복사

⚠ Scope 추가할 때마다 「Reinstall to Workspace」 → **새 토큰 발급** → `.env` 갱신 필요.

## 5. 상대방 식별 (3가지 방법 중 1)

### A. 이메일 (가장 편함)
```bash
python slack_self_delete.py --user hong@회사.com --dry-run
```
> `users:read.email` scope 필요.

### B. Slack 핸들 (@username)
1. Slack 에서 상대방 프로필 클릭
2. 「View full profile」 → 표시명 또는 핸들 확인
```bash
python slack_self_delete.py --user @hong.gildong --dry-run
```

### C. Member ID 직접
1. Slack 에서 상대방 프로필 → **「More actions」 ⋮ → 「Copy member ID」**
2. `U01ABC23DEF` 형식 복사
```bash
python slack_self_delete.py --user U01ABC23DEF --dry-run
```

### D. DM channel ID 직접 (위 3개로 못 찾을 때)
1. Slack 에서 그 DM 열기
2. URL 끝부분이 `D05M91D7LLQ` 형태 → 그게 channel ID
```bash
python slack_self_delete.py --channel D05M91D7LLQ --dry-run
```

## 7. dry-run 미리보기 (실제 삭제 X)

```bash
python slack_self_delete.py --user hong@회사.com --dry-run
```

출력 예시:
```
로그인: user=junsun0708 (U0XXX) team=mycompany
DM 상대: hong@회사.com (U0YYY) → channel=D05M91D7LLQ
대상 채널: D05M91D7LLQ [(DM)]
=== 1단계: 본인 메시지 수집 ===
수집 완료: 247 개
=== 미리보기 (최근 5개) ===
  2024-12-15 14:30  '회의록 공유드립니다'  📎×2
  ... (총 247 개, 첨부 파일 18 개)
```

→ 247 메시지 + 첨부 18개 = 영향 범위 확인 ✅

## 9. 전체 실행 (선택 옵션 조합)

### A. 가장 안전 — 백업 + 첨부 포함 + 병렬
```bash
python slack_self_delete.py --user hong@회사.com \
  --backup --delete-files --workers 5 --execute
```

### B. 기간 지정
```bash
python slack_self_delete.py --user hong@회사.com \
  --since 2024-01-01 --until 2024-12-31 --execute
```

### C. 특정 단어 보존
```bash
python slack_self_delete.py --user hong@회사.com \
  --keep-pattern "계약" --execute
```

### 옵션 조합 정리
| 옵션 | 효과 |
|---|---|
| `--dry-run` | 미리보기만 (기본). 실제 삭제 X |
| `--execute` | 실제 삭제 실행 (필수 명시) |
| `--backup` | 삭제 전 전체 대화 백업 (JSON+TXT) |
| `--backup-only` | 백업만, 삭제 안 함 |
| `--delete-files` | 본인 메시지 첨부 파일도 삭제 |
| `--workers N` | 병렬 worker (1~5 권장) |
| `--since YYYY-MM-DD` | 이후 메시지만 |
| `--until YYYY-MM-DD` | 이전 메시지만 |
| `--limit N` | 최대 N개만 (테스트) |
| `--keep-pattern "단어"` | 그 단어 포함 메시지 보존 |
| `--sleep 1.1` | API 호출 간격 (초) |

## 자주 발생하는 에러

| 에러 | 해결 |
|---|---|
| `missing_scope` | OAuth 페이지에서 scope 추가 → Reinstall → 새 토큰 |
| `invalid_auth` | 토큰 만료/오타 — `.env` 의 `xoxp-...` 확인 |
| `ratelimited` | 자동 백오프 — 그대로 두면 재시도 (최대 5회) |
| `cant_delete_message` | 워크스페이스 정책 — 「Members can delete messages」 비활성 |
| `message_not_found` | 이미 삭제된 메시지 — skip |
| `channel_not_found` | DM 채널 ID 잘못 — `conversations.list?types=im` 으로 확인 |
| `not_in_channel` | (채널 사용 시) 그 채널 멤버 아님 |
| `Pending Approval` | 관리자 승인 대기 — `0-1` 정책 확인 |

## 권한별 시간 예상

| 옵션 | 247 메시지 처리 |
|---|---|
| 기본 (1 worker, sleep 1.1) | ~4분 30초 |
| 5 worker | ~2분 30초 (rate limit 한계) |
| 5 worker + sleep 0.7 | ~2분 (한계 도전) |
| 첨부 18개 포함 | +30초 (각 파일 0.5s) |
